# 웹 업무 자동화 PC 앱 MVP

크롬 확장프로그램 연동 전에 PC 앱의 핵심 사용 흐름을 검증하기 위한 MVP입니다.

## 포함 기능

- 자동화 프로젝트 생성/저장/삭제
- 웹 요소 매핑 수동 등록
- 크롬 확장프로그램 MVP에서 보낸 웹 요소 매핑 수신
- 예시 고객 등록 사이트에서 실제 입력/클릭 실행 확인
- PC 앱에서 확장프로그램으로 실제 열린 웹페이지 실행 명령 전송
- Free, Pro, Business 3단계 요금제 UI
- Free 플랜 프로젝트 2개 제한
- Firebase Auth Google 로그인 및 Firestore 라이선스 동기화
- 값 입력, 버튼 클릭, 대기 단계 구성
- CSV 데이터 불러오기 및 미리보기
- 테스트 실행과 전체 실행 시뮬레이션
- 실행 로그 저장
- 브라우저 `localStorage` 기반 로컬 저장

## 실행

의존성 없이 바로 확인:

```bash
npm.cmd run serve
```

브라우저에서 `http://localhost:4173`을 열면 됩니다.

## Firebase 연결

Firebase는 Google 계정과 라이선스 상태를 저장하는 용도로 연결합니다. 결제 연동 후에는 서버가 Firestore의 `licenses/{uid}` 문서를 갱신하고, PC 앱은 이 문서를 읽어 플랜을 적용합니다.

1. Firebase 콘솔에서 웹 앱을 생성합니다.
2. Authentication에서 `Google` 로그인을 활성화합니다.
3. Firestore Database를 생성합니다.
4. `src/firebase-config.js`의 `YOUR_*` 값을 Firebase 웹 앱 설정값으로 교체합니다.
5. 앱을 새로고침하면 사이드바와 요금제 화면에 Google 로그인 버튼이 표시됩니다.
6. 프로그램 안에서 Google 로그인을 하면 해당 계정 UID로 라이선스를 조회합니다.

라이선스 문서 예시:

```json
{
  "plan": "pro",
  "status": "active"
}
```

Firestore 경로는 `licenses/{Google 로그인 Firebase Auth UID}`입니다.

## 확장프로그램 MVP 연결

1. 크롬에서 `chrome://extensions`를 엽니다.
2. 개발자 모드를 켭니다.
3. `압축해제된 확장 프로그램을 로드합니다`를 누르고 `extension` 폴더를 선택합니다.
4. PC 앱 서버가 켜진 상태에서 원하는 웹페이지로 이동합니다.
5. 확장프로그램 팝업에서 `매핑 모드 켜기`를 누릅니다.
6. 페이지의 입력창이나 버튼을 클릭하면 PC 앱의 웹 요소 매핑 목록에 추가됩니다.

## 실제 웹페이지 실행

확장프로그램을 로드한 뒤에는 PC 앱의 `테스트 실행` 또는 `전체 실행` 버튼으로 실제 열린 탭에 입력/클릭 명령을 보낼 수 있습니다.

1. 자동화할 웹페이지를 크롬에서 엽니다.
2. 확장프로그램 매핑 모드로 입력창과 버튼을 PC 앱에 등록합니다.
3. PC 앱에서 데이터와 자동화 단계를 구성합니다.
4. `테스트 실행`을 누르면 첫 번째 데이터 행만 실제 탭에서 실행됩니다.
5. `전체 실행`을 누르면 CSV 데이터의 모든 행이 실제 탭에서 반복 실행됩니다.

확장프로그램 파일을 수정한 뒤에는 `chrome://extensions`에서 해당 확장프로그램의 새로고침 버튼을 눌러야 최신 코드가 적용됩니다.

Electron 앱으로 실행:

```bash
npm.cmd install
npm.cmd start
```
